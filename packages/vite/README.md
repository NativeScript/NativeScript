## Prerequisites

- NativeScript 9 or higher

## Install

```sh
npm i @nativescript/vite -D
```

## Quick start (`init`)

To bootstrap an existing NativeScript app for Vite, run from your app root:

```bash
npx nativescript-vite init
```

This will:

- Generate a `vite.config.ts` using the detected project flavor (Angular, Vue, React, Solid, TypeScript, or JavaScript) and the corresponding helper subpath from `@nativescript/vite`.
- Add (or update) the following npm scripts in your app `package.json`:
	- `dev:ios`
	- `dev:android`
	- `dev:server:ios`
	- `dev:server:android`
	- `ios`
	- `android`
- Add the devDependencies `concurrently` and `wait-on`.
- Add the dependency `@valor/nativescript-websockets`.
- Append `.ns-vite-build` to `.gitignore` if it is not already present.

After running `init`, you now have two ways to work with Vite:

1. HMR workflow

```bash
npm run dev:ios
```

2. Standard dev workflow (non-HMR)

```bash
ns debug ios --no-hmr
ns debug android --no-hmr
```

## Custom HMR sessions

A NativeScript Vite HMR session may use custom server and staging settings:

| Environment variable | Purpose | Default |
| --- | --- | --- |
| `NS_HMR_PORT` | Vite server port used by the generated device HTTP and websocket URLs | `5173` |
| `NS_VITE_DIST_DIR` | Project-relative staging directory used for Vite output before the NativeScript CLI copies it into the platform app | `.ns-vite-build` |

Leave both variables unset for the standard single-session workflow. Set both
when running concurrent platform sessions for the same app, so the servers and
their platform-specific staged bundles do not collide.

For example, after `npx nativescript-vite init`, use two terminals:

```bash
# Terminal 1: iOS
NS_HMR_PORT=5173 NS_VITE_DIST_DIR=.ns-vite-build/ios \
npx concurrently -k -n vite,ns \
  "vite serve -- --env.ios --env.hmr" \
  "wait-on tcp:5173 && ns debug ios"

# Terminal 2: Android
NS_HMR_PORT=5174 NS_VITE_DIST_DIR=.ns-vite-build/android \
npx concurrently -k -n vite,ns \
  "vite serve -- --env.android --env.hmr" \
  "wait-on tcp:5174 && ns debug android"
```

The environment settings must be visible to both the `vite serve` process and
the `ns debug` process. `NS_HMR_PORT` must also match the port supplied to
`wait-on`. The scripts generated by `init` use `wait-on tcp:5173`, which is
correct for the default session; update that port in custom scripts when
overriding `NS_HMR_PORT`.

A project can keep convenient explicit concurrent scripts, for example:

```json
{
  "scripts": {
    "dev:ios:concurrent": "NS_HMR_PORT=5173 NS_VITE_DIST_DIR=.ns-vite-build/ios concurrently -k -n vite,ns \"vite serve -- --env.ios --env.hmr\" \"wait-on tcp:5173 && ns debug ios\"",
    "dev:android:concurrent": "NS_HMR_PORT=5174 NS_VITE_DIST_DIR=.ns-vite-build/android concurrently -k -n vite,ns \"vite serve -- --env.android --env.hmr\" \"wait-on tcp:5174 && ns debug android\""
  }
}
```

The inline environment syntax above is for POSIX shells. Use the equivalent
environment assignment for the development shell or a cross-platform script
helper when the project must support Windows terminals.

`NS_VITE_DIST_DIR` support requires a NativeScript CLI version that consumes
the same setting when copying Vite output. If the Vite package is upgraded
independently of the CLI, confirm the installed CLI supports this setting
before relying on concurrent output isolation.

## Usage

1) Create `vite.config.ts`:

```ts
import { defineConfig, mergeConfig, UserConfig } from 'vite';
import { typescriptConfig } from '@nativescript/vite/typescript';

export default defineConfig(({ mode }): UserConfig => {
	return mergeConfig(typescriptConfig({ mode }), {});
});
```

Framework-specific configs should be imported from their matching subpaths to avoid loading unrelated framework tooling:

```ts
import { angularConfig } from '@nativescript/vite/angular';
import { reactConfig } from '@nativescript/vite/react';
import { solidConfig } from '@nativescript/vite/solid';
import { vueConfig } from '@nativescript/vite/vue';
```

2) Update `nativescript.config.ts`:

```ts
import { NativeScriptConfig } from '@nativescript/core';

export default {
	// add these:
	bundler: 'vite',
	bundlerConfigPath: 'vite.config.ts',
} as NativeScriptConfig;
```

3) Enjoy Vite.
